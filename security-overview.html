TSC · Security

Security (Common Criteria)

Authorization is enforced in depth. Every request passes through a global, fail-closed authorization pipeline before it can reach application logic.

Access control & authorization

IDControlStatus
AC-01Deny-by-default policyA global fallback policy requires an authenticated principal with an appropriate organizational role on every endpoint; endpoints must explicitly opt in to broader access.Implemented
AC-02Three-tier principal modelCallers are classified as platform admin, organization member, or client tier — derived server-side from validated identity claims, never from client input.Implemented
AC-03Role-based access controlUsers hold role-per-grant assignments (Admin / Editor / Read-Only / Analyst, or org-custom roles) mapped to a granular permission set.Implemented
AC-04Scoped data grantsAccess is constrained to specific organizations, clients, locations, or cities. Effective scope is the union of grants intersected with tier — enforced at the query layer, not just the UI.Implemented
AC-05Write-permission guardA global filter denies all state-changing requests for read-only principals, independent of per-endpoint checks.Implemented
AC-06Least privilege & safeguardsPer-user grant caps and last-administrator protection prevent privilege sprawl and accidental lockout.Implemented
AC-07Controlled impersonationSupport/admin impersonation is validated per request and can only narrow, never widen, data scope. All impersonated actions are attributed to the real actor.Implemented
AC-08Authorization regression testsSnapshot tests fail the build if any endpoint's anonymous, client-allowed, or permission-gated surface changes without an explicit, reviewed update.Implemented

Authentication

IDControlStatus
AU-01Managed identity providerAuthentication is delegated to Clerk. The application validates issuer, token lifetime, and cryptographic signature on every request.Implemented

AU-04Key usage trackingLast-used timestamp and source IP are recorded per key to support anomaly detection and rotation.Implemented

TSC · Confidentiality

Confidentiality

IDControlStatus
CF-01Encryption in transitTLS is enforced end to end; HTTP is redirected to HTTPS; session cookies are Secure, HttpOnly, and SameSite=Strict.Implemented
CF-02Encryption at restData stores run on AWS managed services with encryption at rest.Infrastructure
CF-03Field-level PII maskingPII (email, name, phone, notes, and similar) is masked server-side before being returned to client-tier callers or users lacking PII-view permission. Enforced in the data-mapping layer, not left to callers.Implemented
CF-04Scope enforcement at the query layerConfidential records are filtered to the caller's authorized scope in the database query itself, preventing cross-tenant over-fetching.Implemented
CF-05Secrets handlingRuntime secrets are supplied through environment configuration and the CI/CD secret store rather than embedded in application code paths.Implemented

TSC · Processing Integrity

Processing Integrity

IDControlStatus
PI-01Append-only audit logSensitive mutations — user and access-grant changes, entitlement changes, vendor-credential changes, and key content mutations — are recorded to an append-only trail. The write interface exposes only record and read.Implemented
PI-02Full actor attributionEach entry captures actor, role, any impersonated client, org, action, target, session and request IDs, source IP, and user agent.Implemented
PI-03Defined action vocabularyAudited actions are drawn from a controlled, code-reviewed vocabulary rather than ad-hoc strings, keeping the trail consistent and queryable.Implemented
PI-04Validated, atomic grant changesAccess-grant modifications flow through a single validated, audited service path — direct manipulation of grant records is not permitted.Implemented

Model

Shared responsibility

Dash is responsible for

The security of the platform, its infrastructure, access controls, encryption, monitoring, and the controls described in this document.

Customers are responsible for

Managing their own users and roles, protecting account credentials, enforcing MFA where available, and appropriate use of exported data.
